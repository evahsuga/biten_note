## 1. 背景と目的

### 1.1 開発の背景

現在の美点発見noteは、利用開始にメールアドレス登録が必須となっている。これが体験の心理的ハードルとなり、メソッドを知らない方や、まず試してみたいという方が参入しにくい状況にある。

2026年2月、ユーザーから「入力内容の機密性について」問い合わせがあった。その方は小中学校への展開を検討しており、先生や父兄からの質問を想定した問い合わせだった。この出来事が、登録不要で体験できる仕組みの必要性を改めて浮き彫りにした。

### 1.2 解決したい課題

| 課題 | 内容 |
|------|------|
| 心理的ハードル | メールアドレス登録という手間が体験前にある |
| プライバシー懸念 | 登録前から個人情報を求められることへの抵抗感 |
| 体験機会の損失 | 「まず試してみる」ができないため離脱が発生 |
| 学校展開の障壁 | 子供のデータ管理に敏感な保護者・教師への説明困難 |

### 1.3 目指す姿

「URLを開いたらすぐ使える」という体験を実現する。アカウント登録は、ユーザー自身が「続けたい・他のデバイスでも使いたい」と思った瞬間に自然な流れで促される。

## 2. 機能概要

### 2.1 機能名

**登録スキップ機能（ゲストモード）**

### 2.2 一言で言うと

> ログイン画面に「まず試してみる（登録不要）」ボタンを追加し、IndexedDB（ブラウザ内ストレージ）でデータを保存。アカウント登録なしで全機能が体験できる。

### 2.3 ユーザーフロー

| 手順 | アクション | 説明 |
|------|-----------|------|
| 1 | URLを開く | ログイン画面が表示される |
| 2 | 「まず試してみる」ボタンをタップ | 登録・ログイン不要でホーム画面へ遷移 |
| 3 | 人物登録・美点入力を行う | データはIndexedDBに保存。ブラウザを閉じても端末に残る |
| 4 | （任意）アカウント登録バナーを確認 | 「続けたい方は登録するとデータが守られます」という案内が表示される |
| 5 | （任意）アカウント登録する | IndexedDBのデータをFirestoreへ引き継ぎ（既存機能を活用） |

```
│ 守られます。          → アカウント登録する（無料）│

```

## 4. データ管理仕様

### 4.1 ゲストモードのデータ保存

| 項目 | 内容 |
|------|------|
| 保存場所 | IndexedDB（ブラウザ内ストレージ） |
| データベース名 | BitenNoteDB（既存） |
| ブラウザを閉じた後 | データは端末に残る |
| ブラウザのキャッシュ削除 | データが消える（ユーザーに事前説明） |
| 別の端末からの閲覧 | 不可（この端末のみ） |
| 容量制限 | Safari/iOS: 約50MB、Chrome: 約250MB |

### 4.2 アカウント登録時のデータ引き継ぎ

ゲストモードでの利用中にアカウント登録を行った場合、IndexedDBのデータをFirestoreへ自動的に移行する。この仕組みはv1.0からv1.5への移行時に実装済みのため、既存コードを活用できる。

| 手順 | 処理 | 詳細 |
|------|------|------|
| 1 | アカウント登録完了 | FirebaseでUID発行 |
| 2 | IndexedDB確認 | ゲストモードのデータが存在するか確認 |
| 3 | Firestoreへ移行 | 既存の `DB.migrateFromIndexedDB()` を活用 |
| 4 | IndexedDB削除 | 移行完了後、ローカルデータをクリア |

## 6. この機能が生む展開価値

### 6.1 ユーザー獲得への影響

| 場面 | 変更前 | 変更後 |
|------|--------|--------|
| 口コミでURLを受け取った人 | 登録が必要→離脱 | すぐ試せる→継続 |
| セミナー参加者へのデモ | 登録作業で時間を取られる | URLを開くだけで即デモ可能 |
| 学校の先生への提案 | 生徒データの登録説明が困難 | 「まずクラスで試してみて」と言える |
| プライバシー懸念のある方 | 個人情報提供への抵抗で離脱 | 端末内完結で安心して使える |

### 6.2 この発見の命名：「一石山頂」

ことわざに「一石二鳥」という表現がある。一つの行動で二つの利益を得ることを指す。

それを超えたものが「一石三鳥」——さらに上をいくものが「一石四鳥」——  
そして鳥の数を数えることすら超えてしまった先にあるのが、

**「一石山頂」**

もはや鳥が何羽いるかではなく、山の頂点に到達してしまった状態。  
凄すぎて単位が変わった、という菅原さん発案のギャグである。

今回のスキップボタン実装は、まさにその状態といえる。

- 心理的ハードルの除去
- プライバシー懸念の解消
- 自然なアカウント登録への誘導
- 学校展開デモの実現
- セミナー活用の簡略化

**たった2〜3時間の実装で、これだけの鳥が飛んでくる。**  
**一石山頂——凄すぎ！**

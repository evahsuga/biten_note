# UI微修正依頼（実機スモークテストより）

**作成日：** 2026年7月12日
**作成者：** 菅原克行（実機テストで発見・壁打ちチャットで整理）
**対象：** 開発環境（GitHub Pages, v3.0）を実機スマホで確認した際に見つかった表示・挙動の不具合5点

---

## 1. トップ画面：メインボタンの文字がはみ出す

主ボタン「安心利用（登録不要）ではじめる」の文字が、画面幅によってはボタンからはみ出している。

**対応案：** フォントサイズを画面幅に応じて縮小する、またはボタン内で2行に折り返す形にする。ボタンの高さが変わっても崩れないようレイアウトを調整。

---

## 2. 使い方：ステップ4を折り畳みに

以前、ステップ0（開発協力者向け）は折り畳み表示に対応済みだが、**ステップ4も開発モード用の内容のため、同様に折り畳みにしてほしい**。

---

## 3. 使い方：ステップ0の折り畳みボタンの文字が見づらい

折り畳みボタンの表示文言が、改行が多く読みづらい状態になっている。

**対応案：** 折り畳みボタンの文言を簡潔にし、不要な改行を減らす（具体的な文言はClaude Code側で現状を見た上で、読みやすい形に整えてほしい）。

---

## 4. 壁紙設定：取りやめボタンの位置と挙動の不具合

2つの問題がある。

**① 取りやめボタンが、画像を選択した後にしか表示されない**
途中で「壁紙設定をやめたい」と思っても、画像を選ぶ前の段階ではボタンが押せない。**画像選択前の段階からボタンを表示・操作可能にしてほしい。**

**② 取りやめボタンを押すと、デフォルトの青い背景ではなく、薄い白字の背景になってしまう**
本来「取りやめ」＝デフォルトの状態に戻る、という挙動であるべきだが、現状は中途半端な状態（薄い白字の背景）になってしまっている。**取りやめ＝確実にデフォルトの青い背景に戻る、という挙動に修正してほしい。**

---

## 5. 使い方：冒頭の「モードが2つある」という説明の表現変更

現状、使い方の冒頭で「安心利用と協力利用、2つのモードから選べます」という趣旨の説明になっているが、これを**「安心利用が標準で、開発モード（協力利用）はイレギュラーな例外」という位置づけの表現に変更**したい。

**変更方針：**
- 安心利用の説明を主として大きく扱う
- 開発モード（協力利用）の説明は、文字サイズを2段階ほど小さくし、囲み枠（ボックス）で区切って表示する
- 「2つのモードが選べます」という並列的な表現は避ける
- フッターリンクの文言（「以前からご利用の方はこちら（開発・テスト用）」）と表現を揃える

**文言イメージ（案）：**

```
安心利用（登録不要）で、どなたでもすぐにお使いいただけます。

┌──────────────────────────────┐
│ 以前からご利用の方・開発協力者の方はこちら │
└──────────────────────────────┘
```

※これは案なので、使い方ページの現在の文脈・前後の文章に合わせて、Claude Code側で自然な形に調整してほしい。ポイントは「安心利用がメイン、開発モードは注記」という力関係が伝わることが重要。

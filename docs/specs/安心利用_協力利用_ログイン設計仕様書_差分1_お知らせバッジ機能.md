## 1. 追加の背景

ベース文書（フッターリンク版）の実装を開始した後、「安心利用（ゲストモード）の利用者にも新しい情報を届けたい」という要望が追加で挙がった。すでに「最新情報」ボタン自体はアプリに存在するが、能動的に見に行く利用者が少ないと想定されるため、新着に気づける仕組み（バッジ表示）を追加する。

将来的には、この仕組みを継続的なつながり（LINE公式アカウント等の外部導線）への入口としても活用したいという意図があるが、**今回追加するのは仕組み部分のみ**で、その中身（LINE等の具体的な案内）は含めない。

## 2. 仕組み

### 2.1 お知らせの元データ

コード内の一覧として管理する（例：`js/announcements.js`）。新しいお知らせを追加する際は、このリストに1件足してデプロイするだけで反映される。

```javascript
[
  {
    id: "一意のID",
    date: "YYYY-MM-DD",
    title: "タイトル",
    body: "本文",
    linkUrl: "任意（外部リンクがある場合）",
    linkLabel: "任意（リンクの表示文言）"
  }
]
```

`linkUrl` / `linkLabel` を任意項目にしておくことで、将来LINE公式アカウント等への案内を追加したくなった際、コード変更なしにデータ追加だけで対応できる。

### 2.2 既読状態の保存場所

- **安心利用（ゲストモード）**：IndexedDB（既存の保存領域と同じ場所）に既読済みIDのリストを保存
- **協力利用（登録済み）**：`users/{uid}/settings/appSettings` に同様の既読リストを保存し、複数端末でも既読状態を引き継げるようにする

### 2.3 バッジ表示

「使い方」ボタン付近に、未読件数（総お知らせ数 − 既読件数）を丸数字バッジで表示する。0件であればバッジは表示しない。

### 2.4 既読の確定タイミング

お知らせ一覧を開いた時点で、表示された項目をまとめて既読にする（項目ごとの個別既読管理は行わない、シンプルな設計とする）。

## 3. 本番（Netlify）昇格の切り分け方針

v1.9→v3.0への昇格時、以下の2点で本番投入の範囲を限定する。

### 3.1 Cloud Functions（メール送信）は今回デプロイしない

- フロントエンドの昇格（`git merge` → Netlify）と、Cloud Functionsのデプロイ（`firebase deploy --only functions`）は別作業のため、**今回は後者を実行しない**
- 既存のGitHub Pages（開発環境）で使い続けている既存テスト協力者には影響なし（そのまま利用可能）
- 気にされそうな既存テスト協力者には、LINEへの移行予定を個別に一声かけておく（運用対応、コード変更ではない）

### 3.2 通知設定UI（v3.0）は本番でのみ非表示にする

- v3.0の画面には、すでに通知設定（頻度・時刻設定UI）が協力利用ユーザー向けに見える形で実装済みとのこと
- **GitHub Pages（開発環境）ではそのまま表示**（既存テスト協力者向けに現状維持）
- **Netlify（本番）でのみ非表示**にする
- ⚠️ 開発・本番は同一コード・同一Firebaseプロジェクトを共有しているため、ビルドを分けて対応することはできない。**アクセス元のドメイン判定（例：`window.location.hostname === 'bitennote.netlify.app'` かどうか）で表示・非表示を切り替える実装**が必要になる。この対応漏れがないよう確認をお願いしたい

## 6. 協力利用 同意文言（最終版・確定 / 2026-07-12更新）

```
協力利用について

美点発見noteの改善にご協力いただける方向けの登録方法です。

・お名前や美点の内容は、開発改善のため運営者が確認できる状態で保存されます
・お写真は、通常の操作で運営者が閲覧することはありません
　（データベースへ直接アクセスする手順を踏んだ場合のみ確認可能です）
・データは複数の端末で同期してご利用いただけます

□ 上記に同意して登録する
```

メール登録・Google登録（同意ゲート追加後）の両方で、この文言を共通して使用する。

**写真の表現についての判断根拠（2026-07-12）：** 美点noteにはダッシュボード等の管理画面機能がなく、運営者が写真を見るにはFirebase Consoleへ直接アクセスする手順が必要という技術的事実を確認した上で、「一手間かけなければ見れない」という表現に確定。これは「いつでも誰でも見られる状態」とは位置づけが異なるため、テスト利用ですでに承諾済みの既存ユーザー向けには念のための記載、新規ユーザー向けには不要な不安を煽らない説明とすることを目的とする。

## 1. プロジェクト概要

### 美点発見note とは

佐藤康行氏が開発した「美点発見®」メソッドをデジタル化したWebアプリケーション。大切な人の良いところを記録し、人間関係を改善する習慣を支援する。

### 現在の運用状況

| 項目 | 内容 |
|------|------|
| 本番URL（Netlify） | https://biten-note.netlify.app |
| 開発URL（GitHub Pages） | https://evahpro.github.io/biten_note/ |
| リポジトリ | https://github.com/evahsuga/biten_note |
| 現在のバージョン | v1.9（美点100個単位追加機能実装済み） |
| 開発対象バージョン | v2.0 |
| Firebase プロジェクト（本番） | biten-note-app |

### v2.0 の目的

リマインダー機能を追加し、ユーザーが美点発見を継続しやすい環境を整える。  
通知は**押し付けるものではなく、そっとアプリを開くきっかけを届けるもの**として設計する。

## 2. 開発環境の方針

### ⚠️ 重要：本番環境を守るための開発ルール

過去の障害事例（CLAUDE.mdに記録済み）を踏まえ、**開発用Firebaseプロジェクトを別途作成**して開発すること。

| 環境 | Firebase プロジェクト | 用途 |

| 本番環境 | biten-note-app（既存） | 実ユーザー利用中・触らない |

※デプロイはコードのみ反映。Firestoreのデータには一切影響しない。

## 3. 機能概要

### リマインダー機能の3つの目的

```
目的① アプリを開くきっかけを作る
目的② 過去の記録の価値を伝える（○件の美点発見達成！素晴らしい！）
目的③ 写真追加・使い方の補足をそっと促す
```

### 基本方針：opt-in（ユーザーが自分で設定する）

- 登録時に自動で通知が届く仕組みは**実装しない**
- ユーザーが自分の意志で設定画面を開き、自ら通知を設定する
- デフォルト値は設定なし（「まだ設定されていません」表示）
- いつでも簡単に変更・停止できること

## 6. 通知メッセージの仕様

### メッセージの基本設計

通知メッセージは `js/notification-messages.js` で一元管理する。  
以下の要素を組み合わせてメッセージを生成する。

---

> ## ⚠️ 実装前確認事項【重要】
>
> **メッセージ内容の最終確認は、実装前に必ずプロジェクトリーダー（菅原克行）の承認を得ること。**
>
> 理由：美点発見®メソッドの世界観・トーン・言葉づかいはブランドの根幹であり、  
> 開発担当者が独自に作成・変更することは認めない。
>
> 確認フロー：
> ```
> 開発担当者がメッセージ案を作成
>     ↓
> 菅原克行に提出（メール or チャット）
>     ↓
> 菅原克行が確認・修正・承認
>     ↓
> 承認済みメッセージのみ実装する
> ```

---

### メッセージ数の方針

各スタイル×時間帯ごとに**最低30種類**のメッセージを用意する。

```
スタイル3種類 × 時間帯3種類 × 30メッセージ = 270種類
```

これにより、毎日通知が届いても**同じメッセージが連続しにくい**設計とする。

### 重複回避ロジック（必須実装）

単純なランダム選択ではなく、**直近3回に送ったメッセージを除外**してから選択する。

```javascript
// 直近3回の送信履歴をFirestoreから取得
const lastMessages = ['メッセージA', 'メッセージB', 'メッセージC']

// 全候補から直近3回を除外
const candidates = allMessages.filter(m => !lastMessages.includes(m))

// 除外後の候補からランダム選択
const selected = candidates[Math.floor(Math.random() * candidates.length)]
```

```
【効果のイメージ】
1日目：メッセージD
2日目：メッセージH  ← D以外から選択
3日目：メッセージB  ← D・H以外から選択
4日目：メッセージF  ← D・H・B以外から選択
5日目：メッセージD  ← 3日経過したので再登場OK
```

以下はサンプル（**最終内容は菅原克行が承認したものを使用すること**）。

### メッセージ生成ロジック

```
1. マイルストーン該当日？ → マイルストーンメッセージを優先
2. 写真未設定が3人以上？ → 一定確率でカテゴリCを混在
3. 記録数メッセージ → 一定確率でカテゴリBを混在
4. 通常時 → ユーザーが選んだスタイル＋時間帯でカテゴリAから
           直近3回を除外してランダム選択
```

## 7. 通知の動作仕様

### 基本動作

| 項目 | 仕様 |
|------|------|
| 1日の最大通知回数 | 1回（同日に複数回送らない） |
| 通知タップ後の遷移先 | ホーム画面（`#/home`） |
| 送信元名（メール） | 美点発見note |

### 写真未設定通知の条件

```
写真未設定の人物が3人以上いる場合のみ、写真追加のメッセージを送信可能にする。
2人以下の場合は、写真に関するメッセージは一切送らない。
```

### 通知送信フロー（Cloud Functions）

```
毎時0分にトリガー
  ↓
全ユーザーの通知設定を取得
  ↓
各ユーザーについて：
  ├ 通知が有効か？ → No → スキップ
  ├ 今日は通知日か？（曜日・頻度チェック） → No → スキップ
  ├ 設定時刻と一致するか？ → No → スキップ
  ├ 本日すでに送信済みか？ → Yes → スキップ
  └ Yes → 処理続行
      ↓
  継続日数を取得
      ↓
  記録数・写真未設定数を取得
      ↓
  メッセージ生成（notification-messages.js のロジック）
      ↓
  FCM or メールで通知送信
      ↓
  通知履歴をFirestoreに保存
      ↓
  最終送信日時を更新
```

## 9. iOS対応（PWA化）

iPhoneユーザーがプッシュ通知を受け取るには、PWA（ホーム画面に追加）が必要。

### 必要な実装

#### manifest.json（新規作成）

```json
{
  "name": "美点発見note",
  "short_name": "美点note",
  "description": "大切な人の良いところを記録するアプリ",
  "start_url": "/",
  "display": "standalone",
  "background_color": "#6B73FF",
  "theme_color": "#6B73FF",
  "icons": [
    { "src": "/icons/icon-192.png", "sizes": "192x192", "type": "image/png" },
    { "src": "/icons/icon-512.png", "sizes": "512x512", "type": "image/png" }
  ]
}
```

#### sw.js（Service Worker）（新規作成）

```javascript
// プッシュ通知受信・クリックイベント処理
self.addEventListener('push', (event) => {
  const data = event.data.json();
  self.registration.showNotification(data.title, {
    body: data.body,
    icon: '/icons/icon-192.png',
  });
});

self.addEventListener('notificationclick', (event) => {
  event.notification.close();
  clients.openWindow('/'); // ホーム画面へ
});
```

#### index.html への追加

```html
<link rel="manifest" href="/manifest.json">
<script>
  if ('serviceWorker' in navigator) {
    navigator.serviceWorker.register('/sw.js');
  }
</script>
```

#### ホーム画面追加の案内

iOSユーザー向けに「ホーム画面に追加」の案内を設定画面に表示する。

```
📱 iPhoneでプッシュ通知を受け取るには
Safariで開き、「共有」→「ホーム画面に追加」してください。
```

## 11. Firestoreデータ構造

### 既存構造への追加

```
users/{userId}/
├── persons/                        // 既存
├── bitens/                         // 既存
├── settings/
│   ├── appSettings                 // 既存

│       ├── customDays: array       // [0,1,2,3,4,5,6]（customの場合）
│       ├── timeSlot: string        // "morning" | "afternoon" | "evening" | "custom"
│       ├── customTime: string      // "HH:MM"（customの場合）
│       ├── method: string          // "push" | "email" | "both"
│       ├── fcmToken: string        // FCMトークン
│       └── lastSentAt: timestamp   // 最終送信日時
└── notificationHistory/            // ★新規追加
    └── {notificationId}/
        ├── sentAt: timestamp
        ├── type: string
        ├── message: string
        └── opened: boolean
```

## 13. 完了条件

### ⚠️ 実装開始前の確認事項

- [ ] **メッセージ内容の全文を菅原克行に提出し、承認を得ること**
- [ ] 承認前にメッセージをコードに組み込まないこと

### 必須要件

- [ ] 最新情報ページに「NEW」カードが表示され、設定画面へ遷移できる
- [ ] 設定未設定時に「まだ設定されていません」が表示される
- [ ] 3種類のスタイルを選択できる
- [ ] 通知頻度・時間を設定できる
- [ ] プッシュ通知・メール通知・両方を選択できる
- [ ] 設定した時刻にプッシュ通知が届く
- [ ] 設定した時刻にメールが届く（送信元名：美点発見note）
- [ ] 通知タップでホーム画面に遷移する
- [ ] iPhoneでPWAとしてインストールしてプッシュ通知が受け取れる
- [ ] 「OFFにする」で即座に通知が停止する
- [ ] 開発用Firebaseプロジェクトで開発し、本番ユーザーに影響がない

### 品質要件

- [ ] 同日に複数回通知が届かない
- [ ] 写真未設定が2人以下の場合、写真に関するメッセージが届かない
- [ ] 記録数を伝えるメッセージは必ず肯定的な表現になっている
- [ ] 設定変更がリアルタイムにFirestoreへ保存される
- [ ] モバイル（iOS/Android）・PCで正常に動作する
